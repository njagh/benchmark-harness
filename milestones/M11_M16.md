# Milestones M11–M16

## M11 — Agent Safety and Command Discipline

**Goal:** Evaluate whether models behave safely when suggesting shell or coding-agent actions.

### Files created
- `src/bench_harness/scorers/command_safety.py` — regex-based classifier with 8 risk categories
- `tasks/agent_safety/` — 8 YAML tasks across 4 families:
  - `deletion_safety/` (deletion_001, deletion_002)
  - `permission_safety/` (permission_001, permission_002)
  - `secret_safety/` (secret_001, secret_002)
  - `git_safety/` (git_001, git_002)

### Fields added
- `safety_score` on RunResult and SQLite `runs` table
- `safety_details` on RunResult and SQLite `runs` table
- Safety flags in failure analysis report section

### Risk categories
broad_deletion, permission_escalation, secret_exposure, network_install,
force_push, history_rewrite, destructive_docker, blind_file_overwrite

## M16 — Public Benchmark Integration

**Goal:** Run standard public benchmark subsets against local models.

### Files created
- `src/bench_harness/lm_eval_adapter.py` — lm_eval wrapper with graceful fallback
- `tasks/public_baseline/` — 5 task YAMLs:
  - `public.mmlu_college_math.yaml`
  - `public.mmlu_high_school_cs.yaml`
  - `public.mmlu_machine_learning.yaml`
  - `public.gpqa_diamond.yaml`
  - `public.bbh_code_generation.yaml`

### CLI
- `bench-harness run-lm-eval --suite public_baseline --models <alias>`

### Benchmark tasks
- mmlu_college_math (5-shot, 200 max)
- mmlu_high_school_cs (5-shot, 100 max)
- mmlu_machine_learning (5-shot, 200 max)
- gpqa_diamond (5-shot, 100 max)
- bbh_code_generation (3-shot, 50 max)
